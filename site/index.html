What it does

Session memory that works across every tool

One capture pipeline feeds Obsidian. One MCP server feeds every LLM client. Your context follows you between Claude Code, Hermes, Cursor, and anything that speaks MCP or HTTP.

📝

Auto-capture

Every turn end extracts goals, decisions, files touched, commands run, errors, and TODOs into a structured Markdown note.

🌐

Knowledge graph

Wikilinks connect sessions to shared entities and concepts. Browse the graph in Obsidian's graph view.

🔒

Privacy-first

Extraction is deterministic and offline. Secrets (AWS keys, tokens, JWTs) are regex-redacted before they hit disk.

🔍

Smart recall

BM25 keyword search with optional cosine reranking via ollama embeddings. Your LLM asks, ContextVault answers.

📁

Per-workspace

Memory is scoped to your working directory. Cross-workspace queries are opt-in. No leakage between projects.

🧹

Self-cleaning

10 lint checks catch orphan pages, dead links, stale claims, semantic drift, missing frontmatter, and more.

How it works

Three layers, one vault

ContextVault sits between your LLM client and an Obsidian vault. Everything is plain Markdown — no lock-in, no database.

1

Capture

When a Claude Code turn ends (or a Hermes hook fires), the capture pipeline reads the session transcript, extracts structured facts, redacts secrets, and writes a Markdown note into the vault.

2

Index

A persistent BM25 index updates incrementally on each capture. If ollama is installed, cosine-similarity reranking sharpens the results further.

3

Recall

The MCP server exposes recall, recent_sessions, save_note, and more. Your LLM calls these tools to ground its responses in your past sessions.

Obsidian integration

Your sessions as a navigable graph

Open the vault in Obsidian and press Cmd+G. Sessions, entities, and concepts are connected via wikilinks, forming a knowledge graph you can explore visually.

Get started

Install in under a minute

Clone, init the vault, connect your LLM client. Three commands and you're capturing.

# Install git clone git@github.com:singhdevhub-lovepreet/contextvault.git cd contextvault python3 -m venv .venv .venv/bin/pip install -e ".[dev]" # Put on PATH mkdir -p ~/.local/bin ln -sf "$(pwd)/.venv/bin/contextvault" ~/.local/bin/contextvault # Initialise the vault contextvault init

One command installs hooks + MCP server into Claude Code:

contextvault adapter add claude-code

This merges into ~/.claude/settings.json:

Stop hook — captures the session incrementally after every turn.
SessionStart hook — loads the workspace hot cache into context.
MCP server — registers recall, save_note, recent_sessions, and 3 more tools.

Restart Claude Code to pick up the new settings. That's it — every session is now captured automatically.

Hermes connects via native MCP — no HTTP server needed:

hermes mcp add contextvault --command contextvault --args serve # When prompted "Enable all 6 tools?", type Y # Verify hermes mcp test contextvault # should show "Connected"

For auto-capture (saves every turn to the vault), install the shell hooks described in the README.

Cursor uses the same MCP server — just paste the config:

contextvault adapter add cursor # prints a snippet to paste into ~/.cursor/mcp.json

Restart Cursor. The contextvault MCP server appears under Settings → MCP Tools.

CLI reference

12 commands, zero config

contextvault init Scaffold vault, write config, generate token
contextvault serve Run MCP and/or HTTP server
contextvault capture --cwd . Capture a session transcript
contextvault recall "query" Search the vault, return top-K hits
contextvault lint Run 10 health checks on the vault
contextvault hot Print the workspace hot cache
contextvault workspaces ls List all known workspaces
contextvault export --workspace=WS Zip a workspace for sharing
contextvault adapter add ... Install a client adapter
contextvault ingest FILE Import a file into the vault
contextvault save --title T Pipe stdin into a vault note
contextvault sweep Capture orphan sessions
